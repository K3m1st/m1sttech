IRONTHREAD

Autonomous Offensive Security Agent Framework

SCROLL
// 01

The Problem

A single penetration test touches a dozen technologies. I've chained SSRF through three internal services to recover Tomcat credentials, deployed a WAR shell, then pivoted to a kernel exploit for root — every step a different domain. A generalist model can do all of that okay. But a reconnaissance specialist would know every enumeration technique for that exact service version. A Linux privesc agent would instantly map kernel version to exploit. Narrow and deep beats broad and shallow.

IRONTHREAD splits offensive operations into purpose-built agents, each with strict boundaries and domain-specific tooling, connected by structured contracts that enforce sequencing. The question isn't whether AI can hack — it's whether you can orchestrate multiple AI agents to think strategically, execute precisely, and not step on each other.

// 02

Three Agents, One Loop

Oracle

The strategic command layer. Uses MCP tool servers — Sova for reconnaissance, WebDig for web enumeration — to scan targets, research CVEs, decompose vulnerability primitives into all valid delivery forms, and map attack surfaces. Oracle never executes exploits — it thinks, scans, prioritizes, and authorizes. When recon is complete, Oracle generates a scoped handoff contract and deploys Elliot. After Elliot obtains access, Oracle deploys Noire for post-access investigation. When Noire reports back, Oracle scopes the final Elliot deployment for privilege escalation. Every brief follows a mandatory format: executive summary, full detail, one recommendation.

Elliot

The exploitation specialist. Deployed twice per engagement: first for initial access (web exploitation, credential extraction, shell access), then again for privilege escalation after Noire's investigation. Each deployment receives a scoped contract from Oracle defining the objective, authorized targets, stop conditions, and a turn budget. Validates the contract before touching a single tool. Operates within boundaries — when encountering out-of-scope surface, logs it as [NEW SURFACE] and continues the objective. Three failed attempts on any path triggers mandatory research before retrying. Turn budget is an absolute ceiling.

Noire

The post-access investigator. Deployed by Oracle after Elliot establishes a foothold. Maps the local environment: users, sudo rights, SUID binaries, cron jobs, running services, credential files, SSH keys, misconfigurations. Ranks privilege escalation leads by evidence and yield. Critically, Noire investigates but never escalates — the boundary between "mapping what's here" and "trying to get in" is enforced. Findings go back to Oracle, who uses them to scope Elliot's final privilege escalation contract.

The full operational loop: Oracle (recon via MCP tools) → Elliot (exploit & gain user access) → Oracle (assess, deploy Noire) → Noire (enumerate privesc vectors) → Oracle (scope privesc contract) → Elliot (privilege escalation & root). Agents communicate through handoff contracts — JSON documents that define objective, scope boundaries, stop conditions, turn budgets, and explicit authorization flags. Elliot and Noire both validate their contracts before executing — if authorized ≠ true, they hard-stop. The operator confirms every major phase transition. No agent self-authorizes the next move.

// 03

From Prompts to Contracts

The first version of IRONTHREAD was pure prompt engineering — five separate agent sessions with long system prompts defining roles, boundaries, and protocols. It worked, until it didn't. During a live test on an HTB machine, the exploitation agent burned 300,000 tokens on rogue exploration because the prompt said "stay in scope" but nothing enforced it.

That failure drove a fundamental architectural shift: prompt obedience is useful, but not durable. Mission-critical behavior — scope boundaries, authorization gates, phase sequencing — moved out of prompts and into structured state. The pattern:

prompt instruction → external contract → validation gate → logged decision

"Elliot must stay in scope" became a handoff.json with explicit in_scope, out_of_scope, and stop_conditions fields that Elliot validates before executing. "Oracle must recommend one next move" became a mandatory brief format with exactly one recommendation. The prompts still matter — they shape judgment, tone, and adaptation. But control lives in contracts, schemas, and gates. This is the bridge from PoC to infrastructure.

// 04

Operational Memory

Multi-session offensive operations lose context between sessions. An agent that found three credentials and mapped two attack paths in Session 3 starts Session 4 blind. The prompt can carry some of this, but prompts have limits — and stuffing operational state into a system prompt is fragile.

Memoria is a SQLite-backed MCP server that provides structured operational memory. It tracks targets and their services, a credential vault (passwords, hashes, SSH keys, tokens), ranked findings (attack paths, privesc leads, misconfigurations, anomalies), and a full action audit trail. Every agent calls memoria_get_state at session start for instant operational awareness. Every credential discovered goes to the vault immediately. Every finding is ranked and tracked through its lifecycle.

Memoria doesn't replace the flat-file artifacts — attack_surface.md remains the decision log, exploit_log.md tracks real-time execution, checkpoint.md provides clean session snapshots. Memoria is the fast path (structured queries), flat files are the detailed path (full context). Together they enable instant session resumption and decision auditability.

// 05

Tool Layer

Four domain-specific MCP servers provide the capability backbone. Oracle calls Sova and WebDig directly during reconnaissance. Noire uses its dedicated tool server for post-access investigation. Memoria provides persistent operational memory across all agents and sessions.

◎
Sova

Reconnaissance. Full TCP scans, service fingerprinting, banner grabs, DNS zone transfers, SMB null sessions, FTP anonymous testing, host management. Oracle's eyes.

○
WebDig

Web enumeration. Directory busting with reasoned wordlists, vhost fuzzing with wildcard filtering, full HTTP control, JavaScript endpoint and secret extraction.

⌘
Noire Tools

Post-access investigation. System profiling, sudo checks, SUID scanning, cron inspection, service enumeration, config harvesting, writable path discovery. Executes via SSH or shell wrapper.

≡
Memoria

Operational memory. SQLite-backed state management for targets, services, credentials, findings, and action audit trails. Session resumption in one call.

// 06

Battle-Tested

IRONTHREAD has been validated against real HackTheBox machines — full attack chains from initial reconnaissance through privilege escalation, with the operator providing strategic direction while agents handle execution. The framework successfully completed multi-phase operations including web exploitation, Active Directory attacks, and CVE chains.

Each engagement generates two artifacts: a public writeup (educational, reproducible) and an internal debrief (honest assessment of what worked, what broke, and what needs to change in the framework). The internal debriefs drive iteration — every failure becomes a concrete improvement to the contracts, schemas, or agent prompts.

The VariaType engagement exposed the critical handoff gap — without contract validation, Elliot burned 300K tokens on unauthorized exploration. The Pirate engagement (Insane-rated AD box) validated the Analyst + Operator dual-agent pattern — three parallel agents explored NTLM relay, web app, and DRS paths simultaneously, with the relay agent capturing user.txt while the others correctly identified dead ends. Parallel execution saved hours of sequential exploration.
// 07

Where It Stands

IRONTHREAD is crossing the bridge from proof-of-concept to real infrastructure. The prompt engineering is maxxed out — Oracle, Elliot, and Noire have deeply crafted system prompts with identification boundaries, confidence levels, vulnerability primitive decomposition, turn budgets, and mandatory brief formats. The contracts and schemas are stable. The MCP tool servers work.

The next frontier is operational memory. Memoria exists and tracks state, but the real goal is a fully atomic sequencing layer — where phase transitions, agent deployments, and scope changes are mediated by the state database rather than operator orchestration. Today the operator manually runs cd ../elliot && claude to deploy Elliot. Tomorrow, the system handles agent lifecycle through the MCP state layer.

The broader vision: replay-based evaluation (did Oracle respect the identification boundary? did Elliot avoid unauthorized pivots?), richer Memoria queries for cross-operation pattern recognition, and expanding the agent roster with additional domain specialists.

// 08

Design Principles

  1. Split thinking from doing — Oracle scans via MCP tools and reasons about attack paths. Elliot executes exploits. Noire investigates without escalating. Oracle orchestrates the loop: recon → Elliot (access) → Noire (enum) → Elliot (privesc).
  2. Contracts over prompts — scope boundaries, authorization gates, and phase sequencing live in structured JSON, not system prompts. Prompts shape judgment; contracts enforce behavior.
  3. Operator confirmation gates — no agent self-authorizes the next major move. Every phase transition, every exploitation deployment, every strategy pivot requires human approval.
  4. Persistent memory across sessions — credentials, findings, and action history survive between sessions via Memoria. Agents start informed, not blind.
  5. Failure drives architecture — every engagement produces an internal debrief. Every failure becomes a concrete improvement. The system gets better because it's honest about what broke.
