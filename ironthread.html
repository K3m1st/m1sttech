IRONTHREAD

Autonomous Offensive Security Agent Framework

SCROLL
// 01

The Problem

Pentesting chains multiple technologies — reconnaissance, web enumeration, exploitation, post-access investigation. A generalist model does everything okay. Specialists do each step better. Narrow and deep beats broad and shallow. IRONTHREAD splits the work into purpose-built agents with strict boundaries, connected by structured contracts that enforce sequencing and prevent scope creep.

// 02

Architecture

Oracle

The strategic agent. Reconnaissance, analysis, attack surface mapping. Decides what to attack and why. Builds a prioritized target model and crafts scoped contracts for exploitation.

Elliot

The exploitation agent. Scoped attacks within defined boundaries. Focused, fast, constrained. Receives a contract, executes the attack chain, reports results. No wandering.

Connected by structured handoff contracts — JSON protocols that enforce sequencing, validate signatures, and prevent scope creep. Oracle thinks. Elliot acts. The channel keeps them honest.

// 03

Tool Layer

Four MCP tool servers provide the capability backbone.

◎
Reconnaissance

Network scanning, port enumeration, service fingerprinting. The first look at the target.

○
Web Enumeration

Directory brute-forcing, subdomain discovery, tech stack identification. Mapping the surface.

⌘
Post-Access Investigation

File system traversal, credential harvesting, privilege escalation paths. After the foothold.

≡
Operational Memory

Persistent storage across sessions. Findings, attack history, and learned patterns survive restarts.

// 04

Battle-Tested

Validated against real HackTheBox machines. Full attack chains from recon to root, autonomously. No human intervention between phases.

Early testing exposed a critical gap — without handoff sequencing, agents burned compute on rogue exploration. Elliot would wander off-scope, re-running recon instead of exploiting. The fix — signed contracts with validation gates — is now a core design principle. Failure drove the architecture.
// 05

Design Principles

  1. Split thinking from doing — strategy and execution are separate agents with separate contexts.
  2. Structured handoffs prevent scope creep — JSON contracts with signatures and validation gates.
  3. Persistent memory across sessions — operational knowledge survives restarts and compounds over time.
  4. Human operator in the loop — autonomy within boundaries, escalation when confidence drops.
